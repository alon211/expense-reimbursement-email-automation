# 开发路线图

**最后更新**: 2026-03-11

本文档规划项目未来的开发方向和优先级，帮助团队明确开发目标。

---

## 优先级说明

- **P0（必须）**: 影响项目可用性和可维护性，必须尽快完成
- **P1（重要）**: 显著提升项目功能性和易用性，建议优先完成
- **P2（可选）**: 锦上添花的功能，可根据实际情况安排

---

## P0 优先级（必须）

### 1. 编写 README.md 🚨
**目标**: 帮助新用户快速了解和使用项目

**任务清单**:
- [ ] 项目简介和核心特性说明
- [ ] 快速开始指南（环境配置、运行步骤）
- [ ] 配置说明（.env 文件详解）
- [ ] 常见问题解答（FAQ）
- [ ] 故障排查指南链接

**预计工作量**: 2-3 小时
**依赖**: 无
**负责方**: 待定

---

### 2. 建立单元测试框架 🧪
**目标**: 保证代码质量和重构安全性

**任务清单**:
- [ ] 引入 pytest 测试框架
- [ ] Mock 外部依赖（IMAP、钉钉 API）
- [ ] 编写核心模块单元测试
  - [ ] database.py 测试
  - [ ] rule_loader.py 测试
  - [ ] email_extractor.py 测试
  - [ ] dingtalk_notifier.py 测试
- [ ] 配置测试覆盖率报告（目标 >80%）
- [ ] 集成 CI/CD 自动测试

**预计工作量**: 1-2 周
**依赖**: pytest, pytest-cov, pytest-mock
**负责方**: 待定

**参考资料**:
- [pytest 官方文档](https://docs.pytest.org/)
- [Python 测试最佳实践](https://docs.python-guide.org/writing/tests/)

### 10. 监控告警系统 📊
**目标**: 实时监控系统状态和异常

**监控指标**:
- [ ] 服务健康状态
- [ ] 邮件拉取成功率
- [ ] 规则匹配统计
- [ ] 错误日志聚合
- [ ] 性能指标（CPU、内存、磁盘）

**告警规则**:
- [ ] 服务异常停止
- [ ] 邮件拉取失败率超阈值
- [ ] 磁盘空间不足
- [ ] 数据库连接异常

**技术方案**:
- 使用 Prometheus + Grafana
- 或轻量级方案（Sentry、自研）

**预计工作量**: 1-2 周
**依赖**: 监控系统选型
**负责方**: 待定

## 社区贡献指南

欢迎社区开发者参与项目开发！

### 贡献流程
1. Fork 项目仓库
2. 创建特性分支（git checkout -b feature/AmazingFeature）
3. 提交更改（git commit -m 'feat: Add AmazingFeature'）
4. 推送到分支（git push origin feature/AmazingFeature）
5. 开启 Pull Request

### 贡献方向建议
- 优先实现 P0 优先级任务
- 代码风格遵循 PEP 8
- 单元测试覆盖率 >80%
- 更新相关文档
